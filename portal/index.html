ברוכים הבאים לעולם ה-AI

היכרות
מטרה: הכרות עם בינה מלאכותית, הבנה מה זה ולמה זה חשוב, חוויה ראשונה
🎤 שאלת פתיחה (5 דקות)

למה באתם לקורס? מה אתם רוצים ללמוד? האם באתם להכיר את הכלים ולהנות, או שיש לכם מטרה ספציפית? • מה כבר שמעתם על בינה מלאכותית?

  • מה זה AI? בינה מלאכותית היא כשמחשבים לומדים לבצע משימות שדורשות חשיבה אנושית — הבנת שפה, זיהוי תמונות, קבלת החלטות. בניגוד לתוכנות רגילות שעובדות לפי כללים קשיחים, AI לומד מנתונים
  • סוגי AI: טקסט (ChatGPT, Claude), תמונות (DALL-E, NanoBanana), וידאו (Sora, Novo), קוד (Cursor), מוזיקה (Suno). כל סוג מעבד סוג אחר של מידע
  • AI בחיים היומיום: Siri ו-Google Assistant (קול), נטפליקס (המלצות), Google Translate (שפה), Waze (ניווט), Face ID (זיהוי), פילטרי ספאם (אימייל)
  • הכרות עם ChatGPT ו-Claude: שני הכלים החזקים ביותר. נכנסים, שואלים את אותה שאלה בשניהם, ורואים את ההבדלים — כל כלי חזק בדברים שונים
  • היסטוריה קצרה: הרעיון של AI קיים מ-1950 (אלן טיורינג), אבל הפריצה הגדולה הגיעה ב-2022 עם ChatGPT. ישראל מובילה עולמית בתחום!
  • 💻 מה זה Cursor (קֶרְסוֹר)? Cursor היא תוכנה חינמית לבניית אתרים ואפליקציות בעזרת בינה מלאכותית. במקום ללמוד שפות תכנות — אתם פשוט כותבים מה אתם רוצים, וה-AI כותב את כל הקוד בשבילכם! חשבו על זה כמו "עוזר בנייה חכם" שיודע לבנות אתרים שלמים
  • התקנת Cursor — צעד אחר צעד: (1) היכנסו לאתר cursor.com ולחצו על Download (הורדה). (2) הריצו את הקובץ שירד — EXE בווינדוס, DMG במק. (3) פתחו את Cursor והירשמו עם חשבון Google — הגרסה החינמית מספיקה! (4) לחצו על File → Open Folder (קובץ → פתח תיקייה) וצרו תיקייה חדשה באנגלית בלי רווחים, למשל: my-project
  • איך מפעילים את ה-AI בתוך Cursor? לוחצים Ctrl+K (ווינדוס) או Cmd+K (מק) — ייפתח חלון קטן שבו כותבים מה רוצים לבנות. לדוגמה: "תבנה לי דף נחיתה יפה בעברית עם כותרת, תמונה וכפתור" — ותוך שניות ה-AI יכתוב את כל הקוד! אפשר גם לבקש שינויים: "תחליף צבע לכחול" או "תוסיף עוד כפתור"
  • Claude Code (קלוד קוד) בתוך Cursor: Claude Code הוא ה"מוח" החכם שעובד בתוך Cursor. הוא מבין עברית, בונה אתרים שלמים, מתקן שגיאות ומוסיף עיצוב — הכל לפי מה שתבקשו. כדי להשתמש בו: פתחו את Cursor, לחצו Ctrl+K / Cmd+K, וכתבו את הבקשה שלכם

אמנות הפרומפט

AI
מטרה: ללמוד לדבר עם AI בצורה אפקטיבית + התקנת Cursor ו-Claude Code
🔙 במפגש הקודם

הכרנו את עולם ה-AI, ניסינו ChatGPT ו-Claude, ולמדנו מושגים בסיסיים. • בדיקת משימת בית: מי ניסה 3 כלי AI שונים? מי התקין Cursor? מה גיליתם?

  • מה זה פרומפט? ההוראה או השאלה שנותנים ל-AI. חשבו על זה כמו לדבר עם עוזר חכם מאוד — ככל שתהיו ברורים יותר, כך התוצאה תהיה טובה יותר
  • טכניקה 1 — היו ספציפיים: במקום "כתוב על כלבים" כתבו "כתוב פוסט של 200 מילים על 5 גזעי כלבים מומלצים למשפחות עם ילדים קטנים"
  • טכניקה 2 — תנו הקשר: "אני תלמיד בן 15 שבונה סטארטאפ. עזור לי לכתוב..." — ה-AI צריך לדעת מי אתם ולמה אתם צריכים את זה
  • טכניקה 3 — הגדירו פורמט: "תן לי רשימה ממוספרת / טבלה / 3 פסקאות / נקודות" — תגידו ל-AI בדיוק איך אתם רוצים את התשובה
  • טכניקה 4 — תנו דוגמה: "כתוב בסגנון דומה ל-X, אבל עם Y" — דוגמאות עוזרות ל-AI להבין מה אתם מחפשים
  • טכניקה 5 — שפרו צעד אחר צעד: התחילו פשוט, ואז בקשו שינויים: "תקצר", "תוסיף הומור", "עכשיו תרגם לעברית". כל פעם מקבלים תוצאה טובה יותר!
  • 💻 התקנת Cursor + Claude Code: בחצי השני של המפגש — מתקינים את Cursor (עורך הקוד עם AI) ו-Claude Code (הכלי החכם ביותר לבנייה). ההדרכה המלאה באזור "הדרכות"
Prompt — פרומפט Context — הקשר Iteration — שיפור בשלבים Prompt Chaining — שרשור פרומפטים

למה אותו פרומפט נותן תוצאות שונות בכלים שונים? • מה ההבדל בין פרומפט טוב לפרומפט רע?

🏠 אתגר בית

3. חשבו על 3 בעיות מהחיים שלכם (בית ספר / תחביבים / חברים / משפחה) — תביאו אותן מוכנות למפגש הבא! אלה יהפכו לרעיון הסטארטאפ שלכם.

חשיבה יזמית — מוצאים בעיות

יזמות
מטרה: ללמוד לחשוב כמו יזמים, לזהות בעיות אמיתיות ולהבין את הנוסחה של סטארטאפ מצליח
🔙 במפגש הקודם

למדנו לכתוב פרומפטים מושלמים והתקנו Cursor ו-Claude Code. • בדיקת משימת בית: מי הצליח ליצור לוגו עם AI? מי נרשם ל-Claude Code? הראו את התוצאות!

  • מה זה סטארטאפ? חברה צעירה שפותרת בעיה אמיתית בדרך חדשה או טובה יותר. ההבדל מעסק רגיל: סטארטאפ שואף לצמוח מהר ולהגיע להרבה אנשים
  • סיפורי יזמים ישראליים: Waze (פתרו את בעיית הפקקים), Wix (הקלו על בניית אתרים), Monday (סידרו כאוס בניהול פרויקטים), Iron Dome (ביטחון). ישראל = "אומת הסטארטאפ" — יותר סטארטאפים לנפש מכל מדינה!
  • הנוסחה: בעיה ← פתרון ← מוצר ← עסק. הכל מתחיל מבעיה אמיתית שאנשים רוצים לפתור!
  • Design Thinking (חשיבה עיצובית): שיטה בת 5 שלבים — Empathize/הזדהו (הבינו את המשתמש), Define/הגדירו (מה הבעיה בדיוק), Ideate/חשבו (סיעור מוחות), Prototype/בנו (אב-טיפוס מהיר), Test/בדקו (קבלו פידבק — Feedback, משוב)
  • איך מוצאים בעיות טובות? מה מעצבן אתכם ביומיום? על מה חברים מתלוננים? מה הייתם רוצים שיהיה קל יותר? הבעיות הכי טובות הן אלה שאתם חווים בעצמכם!
  • בעיה טובה vs בעיה רעה: טובה = הרבה אנשים חווים אותה, מוכנים לשלם על פתרון, ספציפית. רעה = עמומה מדי, רק אתם חווים אותה, כבר נפתרה לגמרי
Startup — סטארטאפ Design Thinking — חשיבה עיצובית Validation — אימות בעיה Problem → Solution
💬 שאלות לדיון

מה הדבר שהכי מעצבן אתכם בבית הספר? • איזו אפליקציה הייתם רוצים שתהיה קיימת?

🎯 תרגול כיתה (15 דקות)

סיעור מוחות (Brainstorming): כל תלמיד מזהה 3 בעיות מהחיים שלו (בית ספר, תחביבים, חברים, משפחה) ומציג לכיתה. אחרי ההצגות — הכיתה מצביעה: איזו בעיה הכי מעניינת ושווה לפתור?

🏠 אתגר בית

1. בחרו את הבעיה הכי טובה מתוך ה-3 — זו שאתם הכי מתלהבים ממנה.

2. עשו Validation (אימות): שאלו 5 אנשים (חברים, משפחה) אם גם הם חווים את הבעיה. רשמו את התשובות!

3. מחקר מתחרים (Competitors): חפשו בגוגל — האם כבר קיים פתרון לבעיה? מה הוא עושה טוב ומה חסר בו?

🔗 במפגש הבא נהפוך את הבעיה למוצר אמיתי עם הצעת ערך (Value Proposition) ותוכנית! תביאו את המחקר שלכם.
4

מרעיון למוצר

אסטרטגיה
מטרה: לבחור את הרעיון הטוב ביותר, להגדיר קהל יעד, לבנות הצעת ערך ולתכנן MVP
🔙 במפגש הקודם

למדנו חשיבה יזמית ומצאנו 3 בעיות מהחיים שלנו. • בדיקת משימת בית: מי בחר את הבעיה הטובה ביותר? מי שאל 5 אנשים? מה המתחרים שמצאתם?

  • בחירת הרעיון: מתוך 3 הבעיות מהמפגש הקודם, בוחרים את הטובה ביותר. הקריטריונים: האם הבעיה אמיתית? האם אני מבין אותה? האם אפשר לבנות פתרון? האם אנשים ירצו להשתמש?
  • קהל יעד (Target Audience): מי בדיוק סובל מהבעיה? לא "כולם" אלא "תלמידי תיכון בישראל שמתקשים במתמטיקה". בונים Persona (פרסונה — דמות בדיונית) שמייצגת את המשתמש האידיאלי
  • הצעת ערך (Value Proposition): מה מיוחד בפתרון שלכם? הנוסחה: "אנחנו עוזרים ל-[קהל] לפתור את [הבעיה] על ידי [הפתרון], בניגוד ל-[מתחרים (Competitors)] אנחנו [יתרון ייחודי]"
  • ניתוח מתחרים: מה כבר קיים? חפשו בגוגל, הורידו אפליקציות דומות. מה טוב בהן? מה חסר? ההזדמנות שלכם נמצאת בפערים!
  • MVP (Minimum Viable Product — מוצר מינימלי): הגרסה הכי פשוטה שעדיין שימושית. לא המוצר המלא, רק מספיק כדי לבדוק אם אנשים רוצים את זה
Target Audience — קהל יעד Value Proposition — הצעת ערך Competitors — מתחרים MVP — מוצר מינימלי Persona — דמות משתמש
💬 שאלות לדיון

מי המשתמש שלכם? תארו אותו בפירוט. • מה הדבר האחד שהמוצר שלכם עושה טוב יותר מכל דבר אחר?

🎯 תרגול כיתה (15 דקות)

בונה הפרויקט (Project Builder): מלאו את 10 השאלות המנחות באזור האישי — שם הפרויקט, הבעיה, הפתרון, קהל היעד (Target Audience), והצעת הערך (Value Proposition). זה יהפוך לאפיון (Spec — מפרט) המוצר שלכם!

🏠 אתגר בית

1. השלימו את כל 10 השאלות ב"בונה הפרויקט" — זה האפיון (Specification — מפרט) שלכם!

2. חשבו על עיצוב (Design) האתר: אילו צבעים מתאימים למוצר שלכם? אילו חלקים (Sections — מקטעים) צריכים להיות בדף הנחיתה (Landing Page)?

3. הכינו את הלוגו שיצרתם במפגש 2 — הוא ישמש אתכם בבנייה!

🔗 במפגש הבא נתחיל לבנות את האתר באמת עם Cursor! ודאו ש-Cursor מותקן ועובד.

בנייה

מה זה GitHub Pages?
